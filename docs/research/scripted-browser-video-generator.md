# 脚本驱动浏览器教程视频生成器设计规格

> 创建时间：2026-05-21  
> 状态：设计草案，待用户审阅  
> 后续执行跟踪：如进入实现阶段，再在 `docs/exec-plans/active/` 创建执行计划。

## 目标

构建一个面向编程小白的本地视频生成流水线。用户提供自然语言教程脚本，系统自动拆解为浏览器操作、旁白、字幕和时间轴，随后录制网页操作，生成阿里云 TTS 语音，并通过本地合成工具输出 1080p 横屏教程视频。

MVP 优先解决“清楚、稳定、可定位失败”的教程视频生成，不追求复杂包装、转场或营销感剪辑。

## 推荐方案

采用“CLI 核心 + Skill 包装”的两层方案：

- CLI 核心负责稳定执行：脚本解析、时间轴生成、浏览器录制、TTS、字幕、视频合成和产物管理。
- Skill 包装负责小白体验：用户可直接在对话中粘贴自然语言脚本，由 Agent 追问缺失信息、生成临时脚本文件、调用 CLI，并反馈本地视频路径。
- 后续可在 CodePilot 内增加图形化页面，但不作为 MVP 的必要条件。

## MVP 范围

### 包含

- 输入自然语言教程脚本。
- Agent 将脚本拆解成标准 timeline。
- 浏览器操作必须是明确步骤，例如打开网址、点击文本、输入内容、等待某元素出现、滚动页面。
- 使用 Chromium/Chrome 普通网页环境录制，不优先处理 Electron UI。
- 使用阿里云 TTS 生成中文旁白音频。
- 默认输出 1080p 横屏教程视频。
- 使用本地 `ffmpeg` 合成视频、音频和字幕。
- 输出完整素材包，便于失败排查和二次剪辑。
- 操作失败时立即停止，并输出失败原因、失败段落和页面截图。

### 不包含

- Agent 自主探索网页完成模糊目标。
- 复用用户本机浏览器登录态。
- 竖屏短视频适配。
- 剪映、CapCut、Premiere 等剪辑工具 MCP 集成。
- 自动修复失败操作。
- 高级包装效果，例如转场、局部放大、鼠标高亮、封面、标题卡。

## 用户脚本体验

MVP 面向小白用户，优先支持纯自然语言模板。用户像写教程一样描述每段“说什么、做什么”。

示例：

```markdown
打开产品官网首页。旁白：这一段我们先进入官网，看看首页的主要入口。

点击右上角的“登录”按钮。旁白：接下来点击登录按钮，进入账号登录页面。

在邮箱输入框里输入 demo@example.com。旁白：这里演示如何填写邮箱地址。
```

Agent 负责将脚本转换为内部结构，但不会猜测模糊目标。如果脚本写成“演示如何创建项目”而没有明确点击和输入步骤，执行前应要求用户补充。

## Timeline 结构

内部 timeline 由多个 segment 组成。每个 segment 表示一段旁白和对应的浏览器展示片段。

建议字段：

```json
{
  "segments": [
    {
      "id": "seg-001",
      "narration": "这一段我们先进入官网，看看首页的主要入口。",
      "subtitle": "这一段我们先进入官网，看看首页的主要入口。",
      "actions": [
        { "type": "goto", "url": "https://example.com" }
      ],
      "estimatedDurationMs": 3600,
      "actualAudioDurationMs": 0,
      "bufferMs": 500,
      "assets": {}
    }
  ]
}
```

规则：

- 每段必须有 `narration`。
- 没有操作的段落可作为纯讲解停留段。
- `subtitle` 默认等于 `narration`，后续可支持自动断句。
- `actions` 必须明确到可执行操作。
- 执行前必须完成 timeline 校验，避免录制中途才发现脚本缺失关键字段。

## 时长策略

文字字数只用于预估，最终时间轴以 TTS 真实音频时长为准。

默认建议：

- 教程旁白预估语速：`220 中文字/分钟`，约 `3.67 字/秒`。
- 可配置范围：`180–240 中文字/分钟`。
- 字幕可读性目标：尽量控制在 `4–7 字/秒`。
- 每段展示时长：`max(预估旁白时长, TTS 真实音频时长) + 操作缓冲`。
- 默认操作缓冲：`500ms`，可配置为 `300–800ms`。

依据：

- 专业中文播报语速可达到约 `4.3 字/秒`。
- 教程类内容需要比新闻播报更慢，以便用户同时理解画面操作。
- 中文字幕实务中常以 CPS 控制可读性，中文建议保持在较舒适范围。

## 录制策略

为避免网络加载、页面卡顿或动画等待被录入正片，MVP 采用分段录制，而不是一条连续录屏。

每段流程：

1. 执行本段浏览器操作。
2. 等待页面稳定，例如目标元素出现、网络空闲或显式等待完成。
3. 页面稳定后开始录制该 segment clip。
4. 按最终 segment 时长停留并录制。
5. 停止该段录制，保存 clip。
6. 进入下一段。

这样可以天然排除多余加载等待，最终通过合成阶段拼接所有 clips。

等待策略：

- 每个操作支持显式等待目标，例如等待文本、选择器或 URL 变化。
- 默认单步超时建议为 `15s`。
- 超时后立即停止，不跳过、不继续录制。
- 失败时保存当前页面截图和错误上下文。

## 合成策略

MVP 使用本地 `ffmpeg` 完成合成。

输入素材：

- 分段浏览器录屏 clips。
- 分段 TTS 音频。
- 合并后的完整旁白音频。
- `.srt` 字幕文件。
- `timeline.json`。

输出：

- `final.mp4`：最终 1080p 横屏视频。
- `timeline.json`：最终时间轴，包含真实音频时长和素材路径。
- `subtitles.srt`：字幕文件。
- `run-report.json`：运行报告。
- `clips/`：分段录屏。
- `audio/`：分段音频。
- `screenshots/`：失败或关键检查截图。

字幕 MVP 可先采用烧录进视频的方式，确保用户拿到的视频直接可播放；后续可增加软字幕封装选项。

## 错误处理

MVP 采用早失败策略。

### 脚本解析失败

如果段落缺少旁白、网址、操作对象或操作描述不明确，执行前停止，并提示用户补充。

### 浏览器操作失败

如果找不到按钮、输入框、文本或等待超时，立即停止。报告中必须包含：

- 失败 segment ID。
- 原始脚本文本。
- 失败 action。
- 错误原因。
- 当前页面 URL。
- 当前页面截图路径。

用户可见错误示例：

> 第 3 段的操作“点击 登录 按钮”失败了，页面上没有找到这个文字。请把脚本改成页面上实际显示的按钮文字。

### TTS 失败

阿里云鉴权失败、额度不足、文本过长或接口异常时停止，并保留已生成素材。

### 合成失败

如果 `ffmpeg` 不存在或合成失败，保留录屏、音频、字幕和 timeline，并提示用户安装或修复本地依赖。

## 配置项

MVP 建议支持以下配置：

```json
{
  "viewport": { "width": 1920, "height": 1080 },
  "speechRateCharsPerMinute": 220,
  "segmentBufferMs": 500,
  "actionTimeoutMs": 15000,
  "ttsProvider": "aliyun",
  "subtitleMode": "burn-in",
  "outputDir": "./video-runs"
}
```

阿里云 TTS 凭据应通过环境变量或本地安全配置读取，不写入脚本文件或输出报告。

## 验收标准

MVP 完成后应满足：

- 用户可粘贴自然语言教程脚本，并由 Skill/Agent 调用 CLI 生成视频。
- CLI 可直接读取脚本文件生成同样结果。
- 输出视频为 1080p 横屏 `.mp4`。
- 视频包含网页操作画面、中文语音和中文字幕。
- 语音与字幕基本同步。
- 网络加载等待不会明显出现在最终视频中。
- 浏览器操作失败时立即停止，并输出可定位的截图和报告。
- 输出素材包完整，支持人工排查和二次剪辑。

## 测试方案

- 使用本地 demo 网页测试打开、点击、输入、等待、滚动等基础操作。
- 使用多份自然语言脚本测试 timeline 解析稳定性。
- 使用模拟或真实阿里云 TTS 测试音频生成和真实时长覆盖。
- 使用 `ffmpeg` 合成测试验证最终 `.mp4` 有画面、有音频、有字幕。
- 故意写错按钮文本，验证失败截图和报告内容。
- 人工观看最终视频，确认没有明显加载等待、旁白字幕同步、操作展示清楚。
